Action ID: marketplace/actions/configure-pages

Author: GitHub

Publisher: actions

Repository: github.com/actions/configure-pages

Description

A GitHub Action to enable Pages, extract various metadata about a site, and configure some supported static site generators.

Categories

Deployment Documentation Infrastructure

Categorization

Primary: Deployment

Confidence: high

Reasoning: The action is primarily focused on deploying GitHub Pages, which falls under the Deployment category. Additionally, it involves configuring metadata and settings for a static site, linking it to Documentation and Infrastructure.

Tags: GitHub Pages static site generator deployment configuration infrastructure

Inputs

NameRequiredDescription
static_site_generator No Optional static site generator to attempt to configure: "nuxt", "next", "gatsby", or "sveltekit"
generator_config_file No Optional file path to static site generator configuration file
token Yes GitHub token
enablement No Try to enable Pages for the repository if it is not already enabled. This option requires a token other than `GITHUB_TOKEN` to be provided. In the context of a Personal Access Token, the `repo` scope or Pages write permission is required. In the context of a GitHub App, the `administration:write` and `pages:write` permissions are required.

NameDescription
base_url GitHub Pages site full base URL. Examples: "https://octocat.github.io/my-repo", "https://octocat.github.io", "https://www.example.com"
origin GitHub Pages site origin. Examples: "https://octocat.github.io", "https://www.example.com"
host GitHub Pages site host. Examples: "octocat.github.io", "www.example.com"
base_path GitHub Pages site full base path. Examples: "/my-repo" or ""
