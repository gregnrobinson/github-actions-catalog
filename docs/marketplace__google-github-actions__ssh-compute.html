SSH to Google Cloud Platform compute instances

Action ID: marketplace/google-github-actions/ssh-compute

Author: Google LLC

Publisher: google-github-actions

Description

Use this action to connect to Google Cloud Platform instances via ssh and execute user's commands.

Categories

Infrastructure Authentication

Categorization

Primary: Infrastructure

Confidence: high

Reasoning: The action is primarily used for connecting to Google Cloud Platform compute instances, which falls under Infrastructure. It also involves authentication to establish the SSH connection.

Tags: Google Cloud SSH Compute Infrastructure Authentication

Inputs

NameRequiredDescription
instance_name Yes Name of the virtual machine instance to SSH into.
zone Yes Zone of the instance to connect to.
user No Specifies the username with which to SSH. If omitted, the user login name is used. If using OS Login, USER will be replaced by the OS Login user.
ssh_private_key Yes SSH private key with which to SSH.
ssh_keys_dir No Path for a directory to store ssh keys. Random directory in the temp folder will be generated by default.
container No The name or ID of a container inside of the virtual machine instance to connect to. This only applies to virtual machines that are using a Google Container-Optimized virtual machine image.
ssh_args No Additional flags to be passed to ssh tool. Example: '-vvv -L 80:%INSTANCE%:80'.
command No A command to run on the virtual machine. Action runs the command on the target instance and then exits. You must specify at least command or script, specifying both command and script is invalid.
script No A path for a bash script to run on the virtual machine. Action runs the file on the target instance and then exits. You must specify at least command or script, specifying both command and script is invalid.
project_id No The GCP project ID. Overrides project ID set by credentials.
flags No Space separated list of other compute ssh flags, examples can be found: https://cloud.google.com/sdk/gcloud/reference/compute/ssh/#FLAGS. Ex --ssh-key-expiration=2017-08-29T18:52:51.142Z.
gcloud_version No Version of the Cloud SDK to install. If unspecified or set to "latest", the latest available gcloud SDK version for the target platform will be installed. Example: "290.0.1".
gcloud_component No Version of the Cloud SDK components to install and use. If unspecified, the latest or released version will be used. This is the equivalent of running 'gcloud alpha run' or 'gcloud beta run'. Valid values are `alpha` or `beta`.
