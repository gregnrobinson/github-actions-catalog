Categorization

Reasoning: The action is focused on generating provenance attestations for build artifacts, which is a security measure to ensure the integrity and authenticity of the builds. Additionally, it involves documentation aspects as it generates outputs that describe the attestation.

Tags: provenance attestation security build artifacts

Inputs

NameRequiredDescription
subject-path No Path to the artifact serving as the subject of the attestation. Must specify exactly one of "subject-path", "subject-digest", or "subject-checksums". May contain a glob pattern or list of paths (total subject count cannot exceed 1024).
subject-digest No Digest of the subject for which provenance will be generated. Must be in the form "algorithm:hex_digest" (e.g. "sha256:abc123..."). Must specify exactly one of "subject-path", "subject-digest", or "subject-checksums".
subject-name No Subject name as it should appear in the attestation. Required when identifying the subject with the "subject-digest" input.
subject-checksums No Path to checksums file containing digest and name of subjects for attestation. Must specify exactly one of "subject-path", "subject-digest", or "subject-checksums".
push-to-registry No Whether to push the provenance statement to the image registry. Requires that the "subject-name" parameter specify the fully-qualified image name and that the "subject-digest" parameter be specified. Defaults to false.
show-summary No Whether to attach a list of generated attestations to the workflow run summary page. Defaults to true.
github-token No The GitHub token used to make authenticated API requests.

NameDescription
bundle-path The path to the file containing the attestation bundle.
