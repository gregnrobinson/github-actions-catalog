Inputs

NameRequiredDescription
service-name Yes The name of the ECS Express service. Used for both creating new services and updating existing ones.
image Yes The container image URI to deploy (e.g., 123456789012.dkr.ecr.us-east-1.amazonaws.com/my-app:latest)
execution-role-arn Yes The ARN of the task execution role that grants the ECS agent permission to pull container images and publish logs
infrastructure-role-arn Yes The ARN of the infrastructure role that grants ECS permission to create and manage AWS resources (ALB, target groups, etc.)
cluster No The name of the ECS cluster. Will default to the 'default' cluster.
container-port No The port number on the container that receives traffic (containerPort in primaryContainer). If not specified, Express Mode will use port 80.
environment-variables No Environment variables to set in the container (environment in primaryContainer). Provide as JSON array: [{"name":"KEY","value":"VALUE"}]
secrets No Secrets to inject into the container (secrets in primaryContainer). Provide as JSON array: [{"name":"KEY","valueFrom":"arn:aws:secretsmanager:..."}]
command No Override the default container command (command in primaryContainer). Provide as JSON array: ["node","server.js"]
log-group No CloudWatch Logs log group name for container logs (logGroup in awsLogsConfiguration). If not specified, Express Mode creates a log group automatically.
log-stream-prefix No CloudWatch Logs stream prefix for container logs (logStreamPrefix in awsLogsConfiguration). If not specified, Express Mode uses a default prefix.
repository-credentials No ARN of the secret containing credentials for private container registry (credentialsParameter in repositoryCredentials). Required for private registries outside ECR.
cpu No The number of CPU units to allocate (256, 512, 1024, 2048, 4096, 8192, 16384). If not specified, Express Mode defaults to 256 (.25 vCPU).
memory No The amount of memory in MiB to allocate (512, 1024, 2048, 4096, 8192, 16384, 30720, 61440, 122880). If not specified, Express Mode defaults to 512 MiB.
task-role-arn No The ARN of the IAM role that the container can assume to make AWS API calls (taskRoleArn)
subnets No Comma-separated list of subnet IDs for the service (subnets in networkConfiguration). If not specified, Express Mode uses the default VPC.
security-groups No Comma-separated list of security group IDs for the service (securityGroups in networkConfiguration).
health-check-path No The path for ALB health checks (healthCheckPath). If not specified, Express Mode defaults to /ping.
min-task-count No Minimum number of tasks for auto-scaling (minTaskCount in scalingTarget). Must be less than or equal to max-task-count.
max-task-count No Maximum number of tasks for auto-scaling (maxTaskCount in scalingTarget). Must be greater than or equal to min-task-count.
auto-scaling-metric No The metric to use for auto-scaling (autoScalingMetric in scalingTarget): AVERAGE_CPU, AVERAGE_MEMORY, or REQUEST_COUNT_PER_TARGET
auto-scaling-target-value No The target value for the auto-scaling metric (autoScalingTargetValue in scalingTarget). For example, 60 for 60% CPU utilization.
