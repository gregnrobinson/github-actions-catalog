Inputs

NameRequiredDescription
package-version-ids No Comma separated string of package version ids to delete.
owner No Owner of the repo containing the package version to delete. Defaults to the owner of the repo running the action.
package-name Yes Name of the package containing the version to delete.
package-type Yes Type of package. Can be one of container, maven, npm, nuget, or rubygems.
num-old-versions-to-delete No Number of versions to delete starting with the oldest version. Defaults to 1.
min-versions-to-keep No Number of versions to keep starting with the latest version By default keeps no version. To delete all versions set this as 0.
ignore-versions No Regex pattern for package versions to ignore. Defaults to delete all versions.
delete-only-pre-release-versions No Deletes only pre-release versions. The number of pre-release versions to keep can be specified by min-versions-to-keep. When this is set num-old-versions-to-delete and ignore-versions will not be taken into account. By default this is set to false
delete-only-untagged-versions No Deletes only untagged versions in case of a container package. Does not work for other package types. The number of untagged versions to keep can be specified by min-versions-to-keep. When this is set num-old-versions-to-delete will not be taken into account. By default this is set to false
token No Token with the necessary scopes to delete package versions. If num-old-versions-to-delete is used the token also needs the read packages scope. Defaults to github.token scoped to the repo running the action. To delete package versions of a package outside the repo the action is running in use a Personal Access Token stored as a secret.
