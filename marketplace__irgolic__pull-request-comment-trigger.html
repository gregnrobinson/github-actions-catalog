Pull Request Comment Trigger fork

Action ID: marketplace/irgolic/pull-request-comment-trigger

Author: Unknown

Publisher: irgolic

Repository: github.com/irgolic/pull-request-comment-trigger

Description

Look for a "trigger word" in a pull-request description or comment, so that later steps can know whether or not to run.

Categories

Communication Notification Testing

Categorization

Primary: Communication

Confidence: high

Reasoning: This action is primarily focused on detecting specific comments in pull requests, which falls under communication. It also serves to notify subsequent steps based on the presence of a trigger word, linking it to notification and testing as it influences the workflow execution.

Tags: pull request trigger comment notification workflow

Inputs

NameRequiredDescription
reaction No If set, the specified emoji "reaction" is put on the comment to indicate that the trigger was detected. For example, "rocket".
trigger Yes The string to look for in pull-request descriptions and comments. For example "#build/android"
prefix_only No If 'true', the trigger must match the start of the comment.

Outputs

NameDescription
triggered the string 'true' if the trigger was found, otherwise the string 'false'
comment_body The comment body.
