tofu-remote-state

Action ID: marketplace/dflook/tofu-remote-state

Author: Daniel Flook

Publisher: dflook

Repository: github.com/dflook/tofu-remote-state

Description

Retrieves the root-level outputs from an OpenTofu remote state.

Categories

Infrastructure Dependency management

Categorization

Primary: Infrastructure

Confidence: high

Reasoning: The action retrieves outputs from an OpenTofu remote state, which is primarily related to infrastructure management. It also involves dependency management as it deals with backend configurations.

Tags: OpenTofu remote state infrastructure dependency management GitHub Action

Inputs

NameRequiredDescription
backend_type Yes The name of the OpenTofu plugin used for backend state
workspace No OpenTofu workspace to get the outputs for
backend_config No List of OpenTofu backend config values, one per line.
backend_config_file No List of OpenTofu backend config files to use, one per line. Paths should be relative to the GitHub Actions workspace

Outputs

NameDescription
json_output_path This is the path to all the root module outputs in a JSON file. The path is relative to the Actions workspace. For example, with the OpenTofu config: ```hcl output "service_hostname" { value = "example.com" } ``` The file pointed to by this output will contain: ```json { "service_hostname": "example.com" } ``` OpenTofu list, set and tuple types are cast to a JSON array, map and object types are cast to a JSON object.
