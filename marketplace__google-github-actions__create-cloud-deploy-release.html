Description

Use this action to create a release in Cloud Deploy.

Categories

Deployment Infrastructure

Categorization

Primary: Deployment

Confidence: high

Reasoning: This action is specifically designed to create a release in Cloud Deploy, which falls under the Deployment category, and it also involves infrastructure management within the Google Cloud environment.

Tags: Cloud Deploy Google Cloud Deployment Release Management Infrastructure

Inputs

NameRequiredDescription
name Yes The name of the release.
delivery_pipeline Yes The delivery pipeline to use for the release.
region No Region in which the delivery pipeline can be found.
source No The location of the files to be included in the release.
build_artifacts No Path to a Skaffold output file containing the details of the image(s) to be released.
images No The images to be released.
disable_initial_rollout No Prevent the release being deployed to the first target in the delivery pipeline.
gcs_source_staging_dir No A directory in Google Cloud Storage to copy the source used for staging the build.
skaffold_file No Path of the skaffold file absolute or relative to the source directory.
annotations No Add additional annotations to the release.
labels No Add additional labels to the release.
description No Include a description of the release.
deploy_parameters No Additional parameters to supply at release creation time.
project_id No The Google Cloud Project ID. If unset, this is inherited from the environment.
flags No Space separated list of other Cloud Deploy flags, examples can be found: https://cloud.google.com/sdk/gcloud/reference/deploy/releases/create#FLAGS Example: '--from-k8s-manifest=manifest.yaml --skaffold-version=skaffold_preview'
gcloud_version No Version of the Cloud SDK to install. If unspecified or set to "latest", the latest available gcloud SDK version for the target platform will be installed. Example: "290.0.1".
gcloud_component No Version of the Cloud SDK components to install and use. If unspecified, the latest or released version will be used. This is the equivalent of running 'gcloud alpha run' or 'gcloud beta run'. Valid values are `alpha` or `beta`.
