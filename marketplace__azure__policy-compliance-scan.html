Azure Policy Compliance Scan

‚úì Official Publisher

Action ID: marketplace/azure/policy-compliance-scan

Author: Unknown

Publisher: azure

Repository: github.com/azure/policy-compliance-scan

Description

Triggers compliance scan on Azure resources and passes/fails based on the compliance state of the resources

Categories

Security Monitoring

Categorization

Primary: Security

Confidence: high

Reasoning: The Azure Policy Compliance Scan focuses on assessing the compliance state of Azure resources, which is primarily a security concern, while also monitoring the compliance status of those resources.

Tags: Azure Compliance Security Monitoring Policy

Inputs

NameRequiredDescription
scopes Yes Mandatory. Takes full identifier for one or more azure resources, resource groups or subscriptions. The on-demand policy compliance scan is triggered for all of these. The ID can generally be found in the properties section of the resource in Azure Portal.
scopes-ignore No Optional. Takes full identifier for one or more azure resources, resource groups(followed by /*). If the resources are found non-compliant after the scan completion, the action fails. However, in this input you can specify resources or resource groups for which the compliance state will be ignored. The action will pass irrespective of the compliance state of these resources. In case you want the action to always pass irrespective of the compliance state of resources, you can set its value as all.
policy-assignments-ignore No Optional. Takes full identifier for one or more policy assignments ids. If the resources are found non-compliant for given policy assignment after the scan completion, the action fails. However, in this input you can specify policy assignments ids for which the compliance state will be ignored. The action will pass irrespective of the compliance state of these policies.
wait No Optional. Depending on the breadth, the time taken for compliance scan can range from a few minutes to several hours. By default, the action will wait for the compliance scan to complete and succeed or fail based on the compliance state of resources. However, you can mark this input as false, in which case the action will trigger the compliance scan and succeed immediately. The status of the triggered scan and the compliance state of resources would have to be then viewed in activity log of the resource in Azure portal.
skip-report No Optional. Defaults to false. If false, the action will upload a CSV file containing a list of resources that are non-compliant after the triggered scan is complete. The CSV file can be downloaded as an artifact from the workflow run for manual analysis. Note that the number of rows in CSV are capped at 100,000.
report-name No Optional. The filename for the CSV to be uploaded. Ignored if skip-report is set to true.
