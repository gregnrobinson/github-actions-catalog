Action ID: marketplace/actions/setup-haskell

Author: GitHub

Publisher: actions

Repository: github.com/actions/setup-haskell

Description

Set up a specific version of GHC and Cabal and add the command-line tools to the PATH

Categories

Dependency management Build Infrastructure

Categorization

Primary: Dependency management

Confidence: high

Reasoning: This action sets up specific versions of GHC, Cabal, and Stack, which are essential for managing dependencies in Haskell projects. It also prepares the environment for building Haskell applications.

Tags: Haskell GHC Cabal Stack Dependency Management

Inputs

NameRequiredDescription
ghc-version No Version of GHC to use. If set to "latest", it will always get the latest stable version.
cabal-version No Version of Cabal to use. If set to "latest", it will always get the latest stable version.
stack-version No Version of Stack to use. If set to "latest", it will always get the latest stable version.
enable-stack No If specified, will setup Stack
stack-no-global No If specified, enable-stack must be set. Prevents installing GHC and Cabal globally
stack-setup-ghc No If specified, enable-stack must be set. Will run stack setup to install the specified GHC

NameDescription

stack-path The path of the stack executable _directory_
cabal-store The path to the cabal store
ghc-exe The path of the ghc _executable_
cabal-exe The path of the cabal _executable_
stack-exe The path of the stack _executable_
