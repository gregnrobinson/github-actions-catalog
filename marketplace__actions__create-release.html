Action ID: marketplace/actions/create-release

Author: GitHub

Repository: github.com/actions/create-release

Description

Create a release for a tag in your repository

Categories

Publishing Deployment

Categorization

Primary: Publishing

Confidence: high

Reasoning: The action is focused on creating a release for a tag in a repository, which falls under the publishing category. It also involves deploying the release for users to access.

Tags: release tag GitHub deployment automation

Inputs

NameRequiredDescription
tag_name Yes The name of the tag. This should come from the webhook payload, `github.GITHUB_REF` when a user pushes a new tag
release_name Yes The name of the release. For example, `Release v1.0.1`
body No Text describing the contents of the tag.
body_path No Path to file with information about the tag.
draft No `true` to create a draft (unpublished) release, `false` to create a published one. Default: `false`
prerelease No `true` to identify the release as a prerelease. `false` to identify the release as a full release. Default: `false`
commitish No Any branch or commit SHA the Git tag is created from, unused if the Git tag already exists. Default: SHA of current commit
owner No Owner of the repository if it is not the current one
repo No Repository on which to release. Used only if you want to create the release on another repo

Outputs

NameDescription
id The ID of the created Release
html_url The URL users can navigate to in order to view the release
upload_url The URL for uploading assets to the release
