AWS Lambda Deploy Action

‚úì Official Publisher

Action ID: marketplace/aws-actions/aws-lambda-deploy

Author: Unknown

Publisher: aws-actions

Repository: github.com/aws-actions/aws-lambda-deploy

Description

Updates the code and configuration of AWS Lambda functions. If the function does not exist, a new one will be created.

Categories

Deployment Infrastructure

Categorization

Primary: Deployment

Confidence: high

Reasoning: This action is primarily focused on deploying AWS Lambda functions, which falls under the Deployment category. It also involves managing AWS infrastructure, hence the secondary category of Infrastructure.

Inputs

NameRequiredDescription
function-name Yes Name of the Lambda function.
package-type No The package type of the Lambda function. Set to "Image" for container image functions, defaults to "Zip" for .zip file functions.
image-uri No The URI of the container image in Amazon ECR. Required when package-type is "Image".
code-artifacts-dir No The path to a directory of code artifacts to zip and deploy to Lambda. Required when package-type is "Zip".
handler No The name of the method within your code that Lambda calls to run your function. Required when package-type is "Zip".
runtime No The identifier of the runtime. Required when package-type is "Zip".
s3-bucket No S3 bucket name to use for Lambda deployment package. If provided, S3 deployment method will be used instead of direct upload.
s3-key No S3 key for the Lambda deployment package in the bucket. If not provided, a key will be auto-generated using the format: lambda-deployments/{function-name}/{timestamp}-{commit-hash}.zip.
publish No Set to true to publish a new version of the function after updating the code.
dry-run No Set true to validate the request parameters and access permissions without modifying the function code. Applicable for updating function code only. Creating and updating function configuration will be skipped since they do not support dry run.
revision-id No Update the function only if the revision ID matches the ID that is specified.
architectures No The instruction set architecture that the function supports.
source-kms-key-arn No The ARN of the Key Management Service (KMS) customer managed key that is used to encrypt your functions .zip deployment package.
role No The Amazon Resource Name (ARN) of the functions execution role. Required when creating a new function.
function-description No A description of the function.
memory-size No The amount of memory available to the function at runtime.
timeout No The amount of time (in seconds) that Lambda allows a function to run before stopping it.
vpc-config No For network connectivity to Amazon Web Services resources in a VPC, specify a list of security groups and subnets in the VPC.
environment No Environment variables as a JSON string
dead-letter-config No Specifies the queue or topic where Lambda sends asynchronous events when they fail processing.
kms-key-arn No The ARN of the Key Management Service (KMS) customer managed key
tracing-config No Set Mode to Active to sample and trace a subset of incoming requests with X-Ray.
layers No A list of function layers to add to the functions execution environment.
file-system-configs No Connection settings for an Amazon EFS file system.
image-config No Configuration for the Lambda functions container image.
ephemeral-storage No The size of the functions /tmp directory in MB. The default value is 512, but can be any whole number between 512 and 10,240 MB.
snap-start No The functions SnapStart setting.
logging-config No The Amazon CloudWatch Logs configuration settings for the function.
code-signing-config-arn No The ARN of a code-signing configuration to use on this function.
tags No Tags to apply to the function as a JSON string (e.g. {"Environment":"Production","Team":"DevOps"})
