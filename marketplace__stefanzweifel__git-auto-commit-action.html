Git Auto Commit

Action ID: marketplace/stefanzweifel/git-auto-commit-action

Author: Stefan Zweifel <stefan@stefanzweifel.dev>

Publisher: stefanzweifel

Repository: github.com/stefanzweifel/git-auto-commit-action

Description

Automatically commits files which have been changed during the workflow run and push changes back to remote repository.

Categories

Code quality Automation Dependency management

Categorization

Primary: Code quality

Confidence: medium

Reasoning: This action automates the process of committing changes, which relates to code quality by ensuring that updates are consistently tracked. It also involves dependency management as it handles changes in the repository.

Tags: git auto-commit workflow automation repository

Inputs

NameRequiredDescription
commit_message No Commit message
branch No Git branch name, where changes should be pushed too. Required if Action is used on the `pull_request` event
commit_options No Commit options (eg. --no-verify)
add_options No Add options (eg. -u)

commit_author No Value used for the commit author. Defaults to the username of whoever triggered this workflow run.
tag_name No Tag name used for creating a new git tag with the commit. Keep this empty, if no tag should be created.

push_options No Push options (eg. --force)
skip_dirty_check No Skip the check if the git repository is dirty and always try to create a commit.
skip_fetch No Skip the call to git-fetch.
skip_checkout No Skip the call to git-checkout.

internal_git_binary No Internal use only! Path to git binary used to check if git is available. (Don't change this!)

Outputs

NameDescription
